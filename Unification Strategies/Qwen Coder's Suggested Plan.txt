Qwen Coder's Suggested Plan: Unified PDF Intelligence Architecture
Executive Summary
This document presents a holistic, deeply integrated architecture for the PDF Intelligence Offline application. Rather than simply patching individual issues, this plan reconceives the entire system as a unified text intelligence platform where every component works in concert to deliver exceptional PDF/EPUB upload, indexing, and query capabilities.
The goal is not just consolidation—it's architectural synthesis: weaving together parsing, chunking, tokenization, indexing, search, and UI into a seamless pipeline that maximizes code reuse, minimizes redundancy, and creates emergent capabilities greater than the sum of its parts.

2. Architectural Unification Strategies
2.1 Unified Data Model: The Document Object
Creates typed Document, Chunk, CharPosition, and TokenSpan dataclasses that flow through the entire pipeline, eliminating dictionary passing and enabling type safety.
2.2 Unified Parser Framework: Strategy Pattern
Implements extensible ParserStrategy ABC with PDFParser, EPUBParser, and ParserFactory for easy addition of new formats (TXT, DOCX, HTML).
2.3 Unified Chunking Engine: Adaptive Multi-Strategy
Replaces fragile position tracking with regex-based chunking that supports multiple strategies (paragraph, sentence, fixed-token, semantic) with accurate character offsets.
2.4 Unified Index Manager: Transactional Batch Operations
Introduces atomic transactions via context managers, incremental updates based on file mtime, and clear separation between parsing and storage concerns.
2.5 Unified Search Orchestrator: Two-Phase Pipeline with Caching
Combines query planning, FTS5 retrieval, BM25 ranking, and result caching into a cohesive SearchOrchestrator that returns rich SearchResult objects.
2.6 Unified UI Controller: Event-Driven Architecture
Implements MVC pattern with EventBus, centralized AppState, and ApplicationController that marshals all background operations to the main thread safely.
